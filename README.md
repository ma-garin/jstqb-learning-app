# 試験対策PWAスターター

パートナーと教材をゼロから共同開発するための、GitHub Pages対応の静的PWAフレームです。問題、解説、学習マップ、用語は初期状態では空です。

完成済みのQA学習アプリは別リポジトリへ移動済みです。このリポジトリには、その教材本文や資格固有データを残しません。

## 含まれる機能

- ホームと教材件数表示
- 問題演習、回答、解説、結果表示
- localStorageによる回答済み問題の保存
- 学習マップと用語集
- 学習データ削除
- Web App ManifestとService Worker
- GitHub Pages Actions workflow
- Vitestによるデータ契約・保存処理・静的参照テスト

## 開発

Node.js 20.19以上を使用します。

```bash
npm ci
npm test
python3 -m http.server 8000
```

`http://127.0.0.1:8000/`を開いて確認します。

## 教材の追加

教材データは [js/content.js](js/content.js) の3配列へ追加します。

### 問題

```js
{
  id: 'course-001',
  topic: '自作テーマ',
  question: '独自に作成した問題文',
  choices: ['選択肢A', '選択肢B', '選択肢C', '選択肢D'],
  correctAnswerIndex: 0,
  explanation: '独自に作成した解説'
}
```

### 学習マップ

```js
{
  id: 'topic-001',
  title: '学習テーマ',
  description: 'このテーマで学ぶ内容'
}
```

### 用語

```js
{
  term: '用語',
  definition: '独自に作成した説明'
}
```

## コンテンツルール

- 問題文、選択肢、解説、教材本文は独自作成または明確な利用許諾があるものだけを使う
- 公式問題、書籍、講座、Web教材の転載・翻訳・軽い言い換えをしない
- ロゴ、商標、資格名を使用する場合は、権利と誤認リスクを個別に確認する
- 引用が必要な場合は必要最小限にし、出典と利用条件を確認する
- 秘密情報、個人情報、APIキー、`.env`をコミットしない
- Pull Requestで内容・権利・表示・テストをレビューする

## GitHub Pages

`main`へのpushで `.github/workflows/deploy.yml` が公開対象だけを `_site` に集約して公開します。README、docs、tests、開発用設定はPages成果物へ含めません。リポジトリの `Settings > Pages > Source` は `GitHub Actions` を選択してください。

## Git履歴の扱い

現在の作業ツリーから旧教材を削除しても、過去コミットには旧ファイルが残る場合があります。履歴からも完全に除去する必要がある場合は、共同開発者と合意したうえで次のいずれかを選びます。

- このスターターだけをコピーした新規リポジトリを作成する
- バックアップと影響範囲を確認して `git filter-repo` を使用する

履歴改変は既存clone、Pull Request、コミットURLへ影響するため、無断のforce pushは行いません。
