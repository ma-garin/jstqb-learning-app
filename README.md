# QA基礎学習PWA

ソフトウェアテストとQAの基礎を、現場を想定した独自の説明と問題で学ぶ個人制作のWebアプリです。GitHub Pagesでの公開とPWAとしての利用を想定しています。

## 重要な免責事項

本アプリは、ソフトウェアテスト学習を目的とした個人制作の非公式教材です。

ISTQB、JSTQB、その他関連団体の公式教材、認定教材、認定トレーニングではありません。

学習範囲の確認には、ISTQB/JSTQBが公開する公式シラバスおよび公式情報を参照してください。

本アプリ内の説明、問題、解説は制作者による独自コンテンツです。

ISTQBおよびJSTQBの名称は、参照先や学習範囲を識別するためにのみ使用します。各団体との提携、後援、認定、準拠を示すものではありません。

公式情報:

- [ISTQB公式サイト](https://www.istqb.org/)
- [JSTQB公式サイト](https://jstqb.jp/)

## コンテンツ方針

- 公式PDFをリポジトリや配信物に同梱しません。
- 公式シラバス本文、学習目標文、説明文の転載・翻訳・要約コピーを保存または表示しません。
- 公式サンプル問題の問題文、選択肢、正解、解説を転載・翻案・近似問題化しません。
- 公式ロゴ、公式画像、公式教材に見えるアイコンを使用しません。
- 章番号、節番号、LOコード、Kレベル、資格区分、参照URLは学習範囲を示すメタデータとしてのみ扱います。
- 教材本文、問題文、選択肢、解説は、実務上の一般的な知識を基にゼロから作成します。
- 公式資料は内容の正確性と学習範囲を確認する外部参照先とし、アプリには取り込みません。

## 禁止事項

- ISTQB/JSTQBその他権利者のPDF、ロゴ、画像の再配布
- 公式シラバス本文や公式の学習目標文の転載、翻訳、要約コピー、近似コピー
- 公式サンプル問題、選択肢、解答、解説の転載、翻案、構造の流用
- 公式文を軽く言い換えただけの教材化
- 「公式」「認定」「準拠」など、権利者との関係を誤認させる表示
- アプリ名や説明に公式教材・認定教材と誤認される名称を使用すること

## 問題データの作成ルール

新しい問題は次の形式を基準とします。

```js
{
  id: "qa-basic-001",
  sourcePolicy: "original",
  officialQuestionUsed: false,
  certificationReference: "JSTQB/ISTQB public syllabus referenced only as topic map",
  loCode: "",
  kLevel: "K1",
  topic: "テストの目的",
  question: "自作問題文",
  choices: ["自作選択肢A", "自作選択肢B", "自作選択肢C", "自作選択肢D"],
  correctAnswerIndex: 0,
  explanation: "自作解説。公式文の転載・要約コピーは禁止。",
  reviewerStatus: "draft"
}
```

問題を追加する際は、次をレビューしてください。

- 問題文、選択肢、解説を白紙から独自に作成したか
- 公式サンプル問題と、題材、数値、固有名詞、設問構造、選択肢構成が近似していないか
- 公式シラバスの文章構造や表現をなぞっていないか
- 正解が一意で、誤答にも学習上妥当な理由があるか
- 解説が一般的な実務知識として自立しているか
- 出典を示す場合、本文を取り込まず外部URLだけを掲載しているか
- `sourcePolicy`、`officialQuestionUsed`、`reviewerStatus`が正しく設定されているか
- 公開前に別の人が権利面と内容面を確認したか

## 開発

```bash
npm install
npm test
```

画面は静的HTML/CSS/JavaScriptで構成されています。公開前にはテストに加えて、全ファイル検索、リンク切れ、PWAの表示名、キャッシュ対象を確認してください。

## 過去のGit履歴にPDFがある場合

通常の削除コミットでは、過去のコミットに含まれるファイルは履歴から消えません。公開前に履歴を調査し、公式PDFや再配布できないファイルが過去に含まれていた場合は、次のいずれかを検討します。

1. `git filter-repo`で対象パスを全履歴から削除し、関係者確認後に履歴を書き換える。
2. 安全な現行ファイルだけを使用して、新しい履歴のリポジトリを作成する。

履歴改変はコミットIDと既存cloneに影響します。バックアップ、対象ファイル一覧、共同作業者への告知を行い、リポジトリ管理者の明示的な承認なしにforce pushを実行しないでください。履歴を書き換えた場合でも、既存fork、clone、リリース添付物、Pages成果物、キャッシュから自動的に消えるとは限らないため、個別確認が必要です。
