## 外部資料を参照する際の判断基準

外部資料の英語本文を日本語へ翻訳した学習コンテンツは作成しません。翻訳であっても公式本文の二次利用となり、公式教材との誤認や権利上のリスクを下げられないためです。要約、文章構造の踏襲、用語だけを置換した説明も避けます。

独自教材を作成する際に参照可能な情報は、原則として次のメタデータに限定します。

- 資格区分とレベル名
- 章番号、節番号
- 章・節の短い見出し
- LOコード
- Kレベル
- 資料の版、公開日
- 公式公開ページの外部URL

教材本文は、上記メタデータをトピック地図として使い、一般的なソフトウェア開発・QA実務の知識から白紙で作成します。公式資料を閉じた状態でも説明できる題材、独自の業務例、独自の設問構造を採用し、公開前に公式文・公式問題との近似性を別の人が確認します。

## 履歴対応の判断材料

現行HEADから削除しても過去のGitオブジェクトには残ります。過去のコミットに公式PDF等が存在し、公開リポジトリから取得可能になる場合は、`git filter-repo`による履歴削除または安全なファイルだけを使った新規リポジトリ作成を検討してください。

既存の利用者、fork、cloneが少ない段階では新規リポジトリが明確な選択肢です。既存URLやIssue、PRの継続性が重要な場合は履歴書き換えを検討できますが、管理者承認なしにforce pushを行わないでください。いずれの方法でも外部キャッシュや既存cloneの削除は保証されません。
