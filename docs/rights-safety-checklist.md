# 権利安全チェックリスト

GitHub Pagesへの公開、PWAの配布、リリース作成の前に、現行HEADと実際の配信成果物の両方を確認します。

## 必須確認

- [ ] PDFが含まれていない
- [ ] 公式ロゴが含まれていない
- [ ] 公式シラバス本文が含まれていない
- [ ] 公式問題が含まれていない
- [ ] アプリ名が公式誤認を招かない
- [ ] 非公式表示がある
- [ ] 公式リンクは外部リンクのみ
- [ ] 問題・解説が自作である
- [ ] Google検索されても公式教材に見えない
- [ ] GitHub Pages公開前に全ファイル検索済み

## コンテンツ確認

- [ ] 公式の学習目標文、説明文、表、図を転載していない
- [ ] 公式文の翻訳、要約コピー、語句を置換した近似コピーがない
- [ ] 公式サンプル問題の題材、数値、固有名詞、設問構造、選択肢構成を流用していない
- [ ] 章番号、節番号、LOコード、Kレベル等は参照メタデータに限定している
- [ ] 各問題に独自作成を示すメタデータとレビュー状態がある
- [ ] 教材と解説が公式資料なしでも理解できる独自の実務説明になっている

## 表示と配信確認

- [ ] README、トップページ、設定画面、フッター相当箇所に免責文がある
- [ ] HTML title、manifest、PWA表示名、説明文、OGPに「公式」「認定」「準拠」等の誤認表現がない
- [ ] favicon、PWAアイコン、スクリーンショットが独自デザインである
- [ ] service workerやビルド成果物が削除済みファイルをキャッシュしない
- [ ] 外部リンクには公式情報を別サイトで確認する旨が明記されている
- [ ] `npm test`または既存テストが成功している
- [ ] HTML、JavaScript、CSS、manifest、service workerの参照切れがない

## Git履歴確認

- [ ] 現行HEADだけでなく過去のGit履歴にPDFや公式画像がないか調査した
- [ ] 過去に含まれていた場合、`git filter-repo`による削除または新規リポジトリ作成を判断した
- [ ] 履歴改変が必要な場合、管理者承認、バックアップ、共同作業者への告知を済ませた
- [ ] force pushを無断で実行していない
- [ ] fork、clone、リリース、Pages成果物、キャッシュも個別に確認した
